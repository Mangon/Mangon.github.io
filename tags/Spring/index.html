    很久没有做后端程序的开发，有些生疏了，想要在过年前复习一下Spring框架开发应用后台程序。目前最流行的Java后端应用框架应该就是Spring boot了，网上下了两本书，权且按照书上的流程复习一下整个Spring目前的生态，也自己动手做一个库存管理系统。

    搭建Spring Boot 环境

    首先你需要搭建一个初始的项目架构，大概是下面这个样子

    ├──src // 源代码路径
    │ ├──main // 主要代码
    │ │ ├──java // java代码
    │ │ └──resources
    │ │ ├──static // 静态资源
    │ │ ├──templates // 模板
    │ │ └──application.properties // 应用配置
    │ └──test // 测试相关代码
    ├──target // 编译后代码
    ├──mvnw // 类linux系统打包程序
    ├──mvnw.cmd // windows系统打包程序
    └──pom.xml // 项目maven配置

    这里提供了两种方式

    安装Spring Boot CLI(Spring Boot command-line interface)

    Spring Boot CLI有好几种安装方式。

    • 用下载的分发包进行安装。
    • 用Groovy Environment Manager进行安装。
    • 通过OS X Homebrew进行安装。
    • 使用MacPorts进行安装。

    至此Spring Boot CLI安装成功，你可以运行以下命令行来初始化一个spring boot project

    命令 含义
    -d dependency 依赖
    - -build 编译工具 gradle/maven
    -p package打包形式 jar/war
    -x 解压到当前目录
    -l 查看有哪些可选项

    在STS里创建Spring Boot项目

    1. 在File菜单里选中New > Spring Starter Project菜单项
    2. 填写表单，包括项目名等，点击next
    3. 选择spring boot版本以及依赖，点击next
    4. 确认Spring Initializr请求地址，点击Finish
    5. 在Progress窗口可以看到生成进度

    运行结束后，项目就会显示在STS的explorer中了
